# UXガイドライン・デザインシステム

> 生成日: 2026-07-19（同日、黒板×チョークテイストに全面改定）
> 対象プロジェクト: 漢字ジャストワン（仮）
> ユーザー: 参加者（プレイヤー）のみ。ターンごとに「回答者」「ヒント出題者」の役割が入れ替わる
> 参照元: `docs/requirements/core-requirements.md` / `docs/design/screen-design.md`

---

## 0. デザイン原則

1. **「見せない」を最優先で設計する** — パスプレイの根幹は秘匿性。お題・ヒントは「見てよい状態の確認 → 明示的なタップで表示」を経ないと絶対に見えない。うっかり見えてしまう事故は、ゲーム体験の破壊と同義。
2. **端末を受け取った瞬間に「自分が何をすべきか」が分かる** — 端末は次々に人の手に渡る。各画面には主役のアクションを1つだけ置き、受け取った人が説明なしで操作できるようにする。
3. **漢字が主役。UIは黒子** — 画面の主役はプレイヤーが出した漢字。漢字は大きく・くっきり表示し、UI装飾は漢字より目立たせない。
4. **盛り上がる瞬間に演出を集中する** — 回答者発表・重複削除・答え合わせ・全公開・最終結果は「場が沸く」瞬間。演出はここに投資し、それ以外の操作系画面は軽快さを優先する。
5. **みんなで覗き込める** — 結果系画面（S13〜S15）はテーブル越しに複数人が見る。腕1本分（約50〜60cm）離れても読めるサイズを基準にする。

## 2. タイポグラフィ

### フォントファミリー

| 用途 | フォント | 理由 |
|------|---------|------|
| UI全般・漢字表示（日本語） | Kosugi Maru（Google Fonts / Apache 2.0・アプリ同梱可） | 定番の丸ゴシック。黒板テイストの柔らかさと高い可読性を両立する |
| 数字（スコア・タイマー） | Inter / 等幅数字設定 | カウントアップ時に桁がガタつかない（tabular figures 必須） |

### スケール

| トークン名 | サイズ | 行間 | 用途 |
|----------|-------|------|------|
| `text-xs` | 12px | 1.5 | 注釈・利用規約リンク |
| `text-sm` | 14px | 1.5 | 補助テキスト・設定項目の説明 |
| `text-base` | 17px | 1.6 | 本文（iOS標準に合わせ17px基準） |
| `text-lg` | 20px | 1.5 | リスト項目名・プレイヤー名 |
| `text-xl` | 24px | 1.4 | 画面タイトル・「〇〇さんの番です」 |
| `text-2xl` | 28px | 1.3 | フェーズ見出し・結果画面の小見出し |
| `text-3xl` | 34px | 1.2 | 回答者名の発表・お題の表示（最小） |
| `text-4xl` | 44px | 1.1 | お題表示（推奨）・ラウンドスコア |
| `text-5xl` | 56px+ | 1.0 | ヒント漢字タイル内の文字・最終順位・タイマー |

> **覗き込み画面（S13〜S15）の最低基準**: 漢字・得点は `text-3xl`（34px）以上。50cm離れても読めるサイズ。
> **操作系画面（S03〜S05、S16〜S20）**: `text-base`（17px）基準の通常モバイルUI。

### 文言の漢字ルール

- UIの文言は**小学生が読める漢字**を基準に使い、ひらがな過多にしない（例:「設定」「遊び方」「対戦記録」「回答者発表」）
- それより難しい漢字を使う場合は `<ruby>` で**ふりがな**を付ける
- **お題の表示（S07/S09/S11/S13）には常にふりがなを付ける**（読めないと当てられない・ヒントが出せないため）
- 画面タイトルは「〇〇フェーズ」のような内部用語ではなく、**何をするかが分かる言葉**にする（例: お題を見る・ヒントを書く・ヒントを確認・お題を当てる）

### フォントウェイト

| 用途 | ウェイト |
|------|---------|
| 本文・見出し・漢字（Kosugi Maru） | 400（提供ウェイトは400のみ。強弱はサイズと色のコントラストで付ける） |
| 数字（Inter） | 600〜700 |

## 4. コンポーネントガイドライン

### ボタン

| バリアント | 用途 | 最小サイズ | スタイル概要 |
|----------|------|----------|------------|
| Primary | 進行アクション（次へ・ゲーム開始・決定） | 高さ56px・全幅 | チョーク黄 `#F7C948`・ピル型（角丸=高さの半分）・下端4pxの押し込みエッジ `#C99B1F`・墨ラベル |
| Secondary | 副次アクション（戻る・再確認） | 高さ48px | ボーダーのみ |
| Danger | 手動削除・ゲーム中断 | 高さ48px | 赤系。**必ず確認ステップを挟む** |
| Ghost | 補助導線（遊び方を見る等） | 44×44px | 背景なし・下線 or アイコン |

> **ゲーム進行中（S06〜S15）のPrimaryボタンは画面下部固定・全幅・高さ56px以上**。端末を受け取った人が視線を落とした先（サムゾーン）に必ず次のアクションがある状態にする。

**ステート**:
- `default` / `active`（scale 0.97 + 濃色）/ `disabled`（opacity 0.4）
- パスプレイでは hover は存在しないため、**押下時の視覚・触覚フィードバック（haptic）を必ず入れる**

### 本人確認モーダル（S08 ヒント入力の中核コンポーネント）

パスプレイの秘匿性を守る最重要コンポーネント。ヒント入力のように「特定の1人だけが見てよい」場面で使う。

> S07（お題公開）は端末を回さず全員で一斉に見るため、本人確認は行わない。代わりに同じ見た目の**回答者チェックゲート**（深黒板色・「〇〇さんは見ていませんか？」・OKでお題表示）を使う。

| 要素 | 仕様 |
|------|------|
| 背景 | 全画面を深黒板色（`#1D3A2F`）で覆う。**下のコンテンツは一切透けさせない**（blur不可、完全不透明） |
| 表示内容 | 「〇〇さんに渡してください」→ 渡された後「〇〇さんですか？」 |
| 名前の表示 | `text-3xl`（34px）以上・白文字。誰の番か遠目でも分かる |
| 確認ボタン | 「はい、〇〇です」高さ56px。誤タップ防止のため**表示から0.5秒間は無効化** |
| 否定導線 | 「ちがう人です」はGhostボタンで小さく下部に配置 |
| 表示解除 | 確認後も内容は即表示せず「タップして表示」の明示的操作を1回挟む |

### 漢字タイル（生存文字の表示。S09・S10・S13で使用）

| 要素 | 値 | 備考 |
|------|----|------|
| サイズ | 72×72px以上 | 1文字を Kosugi Maru 44px以上で中央配置 |
| 背景 / 枠 | 白 / `border 2px #E6DFCE` + 濃緑 `#1F4032` のハードシャドウ | 「黒板に貼った漢字カード」のイメージ。border-radius 14px |
| グループ表示（S09） | 同一人物のタイルを枠線で囲む | 囲み枠のみで人物は特定させない。**表示順は毎回シャッフル** |
| フラット表示（S10） | グループ枠なし・等間隔グリッド | 折り返しは中央寄せ |
| 削除済み（S13のみ） | 文字 `#A8A8A8` + 取り消し線 + 背景 `#EDE7D8` | 「自動削除」「手動削除」のラベルを併記。**色だけで区別しない** |

### 手動削除の操作（S09）

「全員一致で削除」は取り返しがつかないため、2段階で設計する。

1. タイルを**長押し（500ms）**で削除候補に（赤枠表示）
2. 「全員が同意したら削除」Dangerボタン → 確認ダイアログ「この文字を削除します。全員同意しましたか？」

### タイマー表示（S07〜S10、任意設定時のみ）

- 画面上部に固定・`text-5xl` 等幅数字
- 残り10秒以下: 警告色（`#F5A623`）→ 残り5秒以下: エラー色 + 1秒ごとのpulse + haptic
- タイムアップしても**強制遷移はしない**（口頭で促す文化を尊重し、アプリは通知のみ）
- iOSの「視差効果を減らす」設定時はpulseを停止

### スコアボード・順位表示（S14・S15）

- 1位: 金 `#D9A62E` + 王冠アイコン / 2位: 銀 `#9FA8B0` / 3位: 銅 `#B0714A`
- 順位・名前・得点を1行に。行の高さ56px以上、得点は `text-2xl` 等幅数字
- 同点は同順位で表示する
- S15（最終結果）は1位の発表を演出付きで最後に見せる（下位から順に表示）

### 入力フィールド（S08 ヒント入力・S10 回答入力）

| 要素 | 仕様 |
|------|------|
| 高さ | 56px以上・`text-2xl`で入力文字を大きく表示 |
| キーボード | iOS標準の日本語キーボード。**確定した漢字のみ受け付ける**（ひらがな・カタカナ・英数は送信ボタンを無効化しインラインで理由を表示） |
| 文字数 | 設定文字数に達したら残り0を明示（「あと1文字」カウンタ表示） |
| フォーカス時 | border 2px `#4A4A44`（チョーク枠が濃くなる表現） |
| エラー時 | border 2px 赤 + 下部にエラーテキスト |

## 6. アニメーション・トランジション

| 種別 | 時間 | イージング | 用途 |
|------|------|-----------|------|
| マイクロインタラクション | 150〜200ms | ease-out | ボタン押下・タイル選択 |
| 画面遷移 | 250〜300ms | ease-in-out | フェーズ切り替え |
| 回答者発表（S06） | 800〜1200ms | ease-out | ルーレット/ドラムロール的に名前を表示（毎回選択モード時） |
| 重複削除の演出（S09表示前） | 600〜900ms | ease-in | 被った文字が「消える」演出。このゲーム最大の見せ場のひとつ |
| 正解フィードバック（S11） | 400〜600ms | ease-out | 明るいバースト + 効果音 |
| スコアカウントアップ | 800〜1200ms | ease-out | 加点・ボーナス加点を順に積む |
| 全公開（S13） | 1タイル100ms間隔 | ease-out | 誰が何を出したかを順にフリップ表示 |
| 最終結果（S15） | 下位から順次表示 | ease-out | 1位発表時のみ紙吹雪 |

**演出ガイドライン**
- 不正解・ギブアップ時は控えめに（短いフェード。責める演出・重い効果音は使わない）
- 操作系画面（S03〜S05・S16〜S20）では演出を入れず軽快さを優先する
- iOS「視差効果を減らす」（Reduce Motion）設定時は全演出をフェードに置き換える
- 効果音はS20の設定でOFFにできる。**演出は音がなくても成立するように作る**（公共の場・深夜プレイを想定）

## 7. レイアウト・グリッド

### 基本（スマホ縦向き基準）

- 1カラム・全幅。ページ左右padding 16px
- 最大幅: 480px（iPad等ではセンタリング）
- セーフエリア（ノッチ・ホームインジケータ）を必ず尊重。下部固定ボタンは `safe-area-inset-bottom` + 16px
- 横向き（ランドスケープ）は v1 では非対応（縦固定）。対面での覗き込み対応は将来検討

### ゲーム進行画面（S06〜S15）の共通構造

```
┌─────────────────────┐
│ やること・進行状況（n人目・回答者名）│ ← 上部固定・text-sm
│                     │
│      メインコンテンツ        │ ← 漢字・名前・結果（主役）
│      （余白広め＝舞台）       │
│                     │
│ ［ 次へ進むPrimaryボタン ］    │ ← 下部固定・全幅・56px
└─────────────────────┘
```

- 上部に進行状況を「ラウンド n/N ・ m/M人目 ・ 回答者: 〇〇」の形式で常時小さく表示する（「ターン」表記は使わない）。端末を受け取った人が状況を即把握できる
- 画面は**3層ゾーン**で構成する: ①ヘッダー帯（ステータスバー＋フェーズ情報。`#24483A`）②作業ボード（黒板緑。このフェーズでやることだけを置く）③操作帯（下部固定ボタン領域。`#24483A`）。帯と板の境界はチョーク罫線 `#4E7561` 2px で区切り、領域の役割が地色だけで区別できるようにする

## 8. ナビゲーション方針

### ホーム系（S01〜S05・S16〜S20）

- S01をハブとする標準的なスタック遷移。左上に戻るボタン
- 買い切り導線（S04→S19）は錠前アイコン + 「解除する」で統一し、押し付けがましいモーダル連発はしない

### ゲーム進行中（S06〜S15）

- **戻る操作は原則封じる**。スワイプバック無効化・戻るボタン非表示。誤操作でゲーム状態が壊れることを防ぐ
- 中断したい場合は右上のメニュー（…）→「ゲームを中断する」→ 確認ダイアログの2段階のみ
- アプリがバックグラウンドに落ちても**ゲーム状態は必ず保持**し、復帰時に同じフェーズから再開できる
- バックグラウンド移行時、秘匿情報（お題・入力中ヒント）が表示中なら**Appスイッチャーのスナップショットを隠す**（プライバシーオーバーレイ）

## 9. エラー・空状態・ローディング

### エラーメッセージ

- ローカル完結のゲームのため通信エラーはほぼ購入フロー（S19）のみ。「購入を完了できませんでした。通信環境を確認して、もう一度お試しください」のように**原因 + 次の行動**を必ず添える
- 入力バリデーション（S08/S10）はインライン表示。「漢字のみ入力できます」など短く

### 空状態

| 場面 | 表示 |
|------|------|
| 履歴0件（S16） | 「まだ対戦記録がありません」+「ゲームを始める」CTA |
| 自作お題0件（S05） | 「オリジナルのお題を作ってみましょう」+ 追加ボタン |
| プレイヤー未登録（S03） | 新規登録フォームをそのまま表示（空リストを見せない） |

### ローディング

- ローカル処理主体のためローディングは原則発生させない（100ms以内に応答）
- 購入処理（S19）のみボタン内スピナー + 二重タップ防止
- 演出の「タメ」（重複削除・結果発表）はローディングではなく意図的なアニメーションとして設計する
