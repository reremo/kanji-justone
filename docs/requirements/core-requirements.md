# 漢字の力要件定義 v1

## 概要

1台のiOSデバイスを複数人で回して遊ぶ、パーティゲーム。回答者以外の参加者が、お題に関するヒントを漢字で出し合い、回答者がお題を当てる。他の出題者と漢字が被ると無効になるため、被らない漢字を選ぶ駆け引きが core loop。

## 対象プラットフォーム・デバイス

- iOS アプリ
- 1台のデバイスを回して遊ぶローカルパスプレイ専用（v1）
- オンライン対戦・複数端末連携は将来検討事項。実装制約が大きい場合は見送る

## ゲームルール

### 1ターン（1つのお題）の流れ

1. **お題決定**：通常お題プールからランダム抽選、または「自作お題専用モード」の場合は自分たちが登録したリストから抽選
2. **お題公開フェーズ**：回答者が画面を見ていないことを全員で確認したうえで、回答者以外の全員が**一斉に**画面を覗き込んでお題を確認する（端末を順番に回す必要はない）。全員が見終わったら、誰でもボタン一つでお題を引き直す（スキップ）ことができる。回数制限やトリガー権限の制約は設けない
3. **ヒント入力フェーズ**：ヒント出題者に順に端末を回し、都度「〇〇さんですか？」の確認を挟んで各自が漢字を入力する（基本2文字。設定で1人あたりの文字数を変更可）。入力UIは**1文字ごとに独立した入力枠**とし、2文字で1つの熟語を入れるものと誤解させない。**同一人物が同じ漢字を複数回出すことは入力時に禁止**する（インラインエラーで弾く）
4. **自動重複削除**：全員の入力が終わったら、**1文字単位**で重複している文字をシステムが自動的に取り除く（例：Aさんの「山川」とBさんの「山風」→ 双方から「山」が消え、「川」「風」だけが残る）
5. **ヒント確認フェーズ**：ヒント出題者全員で、お題と生き残った文字を確認する（画面にお題も表示する）。
   - 同一人物が出した漢字は**グルーピング表示**する（「誰が」は分からないが「同じ人物の組」だと分かる形）。これは「同一人物の2文字で実在の言葉を作っていないか」を出題者自身がチェックできるようにするため
   - グループの表示順は**毎回ランダムにシャッフル**する（提出順・端末を回した順とは無関係にし、位置から誰の漢字か推測できないようにする）
   - 自動重複削除で消えた文字は、文字そのものは表示しないが、**「消えた枠」として消えた感のある表現で残す**（何文字消えたかは分かる。どの文字かはターン結果まで伏せる）
   - ルール違反（お題に含まれる漢字・和訳した漢字やその一部の使用、同一人物の2文字で実在の言葉を作っている、等）に気づいた場合、**全員一致で手動削除**できる。この違反判定自体はアプリは行わず、人力で行う
   - 削除の結果、生存漢字が**0になった場合は強制ギブアップ**（全員0点）とし、専用演出で「全滅」を盛り上げる。回答フェーズ以降はスキップする
6. **回答フェーズ**：「回答者に渡してください」の受け渡し画面を挟んで回答者に端末を渡す（生き残った漢字は誰が見てもよい情報のため、本人確認は不要）。生き残った漢字は**フラット**に見せる（グルーピングなし、順不同）。削除された文字の分は「消えた枠」として表示する（文字は見せない）。回答者がお題を入力する
7. **答え合わせ**：ヒント出題者に見せて正誤判定する
   - **不正解**の場合：回答フェーズに戻って再回答できる（回数無制限）。諦める場合は明示的にギブアップを選ぶ
   - **正解**の場合：回答者に得点を付与し、ヒント出題者にはヒント採点フェーズの結果に応じて得点を付与する（最高得点はどちらも**+参加人数**）。さらに、**回答者にのみ**自動削除＋手動削除の合計削除数分のボーナスを加点する（少ないヒントで当てたことへの報酬）
   - **ギブアップ（正解できない）**の場合：全員0点。採点はスキップし、ターン結果（誰が何を出したか）の表示は行う
8. **ヒント採点フェーズ**（正解時のみ）：回答者が生き残った漢字を「役に立った順」に順位付けする。得点は漢字単位ではなく**出題者単位**で決める：各出題者の生存漢字のうち**最も順位の高い漢字**の順位で出題者を並べ、その順に高い得点を与える（1位は+参加人数、以下1点ずつ減）。例：4位の漢字1枚しか生存していない出題者は最下位になる。**生存漢字が0枚の出題者は0点**（順位付けの対象外）。この計算過程や配点の詳細はUIには表示しない
9. **ターン結果**：誰がどの漢字を出したか、自動・手動で削除された漢字も含めて全て公開し、このお題を終了する。画面にはお題を大きく表示し、各漢字の順位（役に立った順）と各プレイヤーに与えられた点数を明記する（UI名称は「全公開」ではなく「結果」とする）

### ラウンドとゲームの構造

- 参加者全員が1回ずつ回答者を経験する1周を**1ラウンド**と呼ぶ
- ラウンド数（周回数）はゲーム開始時に選択できる
- 回答者の決め方は3択:
  - **じゅんばん**：参加登録順の固定ローテーション
  - **ランダム**：完全ランダムではなく**ラウンドロビン**。1ラウンド内で全員が1回ずつ回答者になることは保証し、その順序だけをラウンドごとにシャッフルする
  - **固定**：特定の1人がずっと回答者を務める（この場合の「1ラウンド」は**回答者以外の人数分のターン**とする）
- 合計得点で勝敗を決める
- ラウンド終了時の中間スコアには、合計点に加えて前ラウンドの結果（前回の順位・そのラウンドの獲得点）も表示する

### 設定項目

- 参加人数：3〜8人（推奨3〜6人）。2人ではヒントの重複削除・匿名グルーピングが成立しないため下限は3人とする
- 1人あたりの漢字文字数：**無料は参加人数で自動決定**（3人=3文字・4人=2文字・5人以上=1文字）で変更不可。**買い切りで1〜5文字を自由に設定できる**。いずれも「その文字数で実在する言葉を作らない」というルールは維持する
- ラウンド数：**1〜10ラウンド**。無料は上限2ラウンド（買い切りで10まで）
- 難易度：易・普通・難の3段階。**ラウンド単位**で固定し、ラウンド中に難易度は変わらない（ラウンドごとに難易度を変えることは可能）。難易度は通常お題の設定項目であり、設定UI上はお題モードと同一項目にまとめる。**自作お題モード選択時は難易度を表示しない**

### 中断・再開

- プレイ中は常時「一時中断」メニューを開ける。メニューでは以下ができる:
  - サウンドのON/OFF
  - 中断してホームへ戻る（進行を保持し、あとで再開できる）
  - ゲームをやめる（進行を破棄。確認ダイアログを挟み、記録には残さない）
- 中断中に「ゲームを始める」を押すと「再開する／新しく始める」の選択モーダルを表示する
- v1の既知制約: 中断データはアプリ起動中のみ保持し、アプリの完全終了で消える（永続化はSwiftData移行時に対応・TBD）

## 参加者管理

- 参加者（プレイヤー）リストは端末に保存する
- 次回以降のプレイでは、リストから選択するだけで参加できる（毎回名前を手入力する必要はない）

## お題（コンテンツ）

### 通常お題

- 3段階の難易度 × カテゴリ。**各難易度100問（合計300問）**を用意する
- **全お題にふりがなデータを付与する**（お題表示は常にふりがな付きのため）
- 制作方法：AI（LLM）で下書きを生成し、人間が目視チェック・修正して確定させる
- 難易度の振り分けは制作時の感覚でざっくり行う（厳密な線引き基準は設けない）
- 無料開放はやさしい30/ふつう20/むずかしい0。むずかしいは全問が買い切り限定になる

### お題一覧

- ホームの「お題一覧」から、通常お題と自作お題をタブで一覧閲覧できる
- 通常お題タブでは難易度（易・普通・難）もタブで切り替えて表示する
- 無料版で未開放の通常お題は内容を伏せて表示する（買い切りへの導線）

### オリジナルお題（自作モード）

- ユーザーが自由テキストでお題を登録できる（バリデーションなし）
- 自作お題の管理（追加・編集・削除）はお題一覧画面の自作タブで行う。独立した管理画面は設けない
- 通常お題の抽選プールには混ぜず、「自作お題専用モード」として別立てで提供する（自分たちが登録したリストからのみ出題される）

## 履歴・統計

- v1から簡易な履歴・統計画面を用意する
  - 過去の対戦履歴（日時・参加者・得点など）
  - 個人の集計データ（例：有効な漢字をどれだけ出せているか、など）
  - 個人統計には「どのお題のときに、なんの漢字を出したか」の履歴を表示する（削除された漢字も区別して表示）
- 裏側では「誰が・どのお題に対して・どの漢字を出したか」を記録し、将来的なAI分析（その漢字で正解できたかの予測、性格診断など）のためのデータ基盤とする。ただし**AI分析機能自体はv1のスコープ外**（将来バージョンで実装）

## マネタイズ

- 無料版：お題数・ラウンド数上限・ヒント漢字数（1人あたりの文字数変更）を制限し、広告を表示する。**参加人数は制限しない**
- 無料版の制限値（お題）：**やさしい30問・ふつう20問・むずかしい0問**を無料開放（残りは買い切りで解放）。ラウンド上限2・ヒント文字数は人数で固定（3人3/4人2/5人以上1）
- 設定UIで無料枠を超えるラウンド数を選ぼうとした場合、その「＋」ボタンにプレミア（🔒）を表示してショップへ誘導する。上限10を超える操作は通常のグレーアウト
- App Store は一般カテゴリ（ゲーム/ワード）・**対象年齢9+**で配信し、Kidsカテゴリには入れない。広告は**非パーソナライズ**で開始する（ATT/パーソナライズは後から検討）
- 買い切り：単一の「全解除」商品1種類のみ（広告除去＋ラウンド数を10まで解放＋ヒント文字数を自由設定＋通常お題全解放＋自作お題）。DLC的な追加お題パックの個別課金などは設けない
- オリジナルお題（自作モード）は買い切り限定機能
- 広告（無料版のみ）は2種類：①フッター常設のバナー広告 ②**ラウンド結果・最終結果の後**に自動表示されるインタースティシャル広告（全画面・スキップ可・報酬なし）

## v1スコープ外

- AI分析機能（正誤予測・性格診断）
- Game Center連携（実績・リーダーボード）
- オンライン対戦・複数端末連携
- 買い切り以外の追加課金（お題パック等の個別販売）
